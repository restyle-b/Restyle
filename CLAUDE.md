# CLAUDE.md — קובץ מנחה לפרויקט Restyle

קובץ זה מנחה את Claude Code בעבודה על הפרויקט. קרא אותו בתחילת כל סשן.

## מהו הפרויקט
אתר פרימיום למספרה **Restyle** הכולל:
- **אתר תדמית** — דף בית, אודות, שירותים, גלריה, צור קשר (השראה: https://menspire.com — נקי, מינימליסטי, פרימיום, תמונות גדולות).
- **חנות** — מכירת מוצרים אונליין עם עגלה, צ'קאאוט ותשלום.
- **אקדמיה** — קורסים, דפי קורס, הרשמה.
- **אזור אישי** — התחברות, פרופיל, היסטוריית הזמנות, גישה לקורסים, תורים.
- **ניהול הזמנות** — מחזור חיים מלא של הזמנה.
- **ממשק ניהול (Admin)** — ניהול מוצרים, הזמנות, קורסים, תורים, תוכן ומשתמשים.
- **תשתית תשלומים** — שכבת הפשטה שתחובר בעתיד ל-Tranzila / HYP.

## הסטאק (מאושר)
- Next.js 15 (App Router) + React 19 + TypeScript strict
- Tailwind CSS + shadcn/ui, **RTL ועברית כברירת מחדל**
- PostgreSQL (Supabase) + Prisma
- Supabase Auth
- Cloudflare R2 לאחסון תמונות (S3-compatible)
- תשלומים דרך interface `PaymentProvider` (Tranzila/HYP בעתיד)
- אירוח: Vercel
- בדיקות: Vitest + Playwright

פירוט מלא ונימוקים: ראה `docs/ARCHITECTURE.md`.

## הסקילים — מתי להשתמש
חמישה סקילים מותאמים לפרויקט ב-`.claude/skills/`:
1. **planning** — לפני כל פיצ'ר/שלב חדש: מודל נתונים, חוזה API, מסכים, קריטריוני קבלה.
2. **development** — בכתיבת/שינוי קוד: תקני קוד, מבנה, סטאק.
3. **qa** — אחרי פיתוח, לפני "בוצע": בדיקות מול קריטריוני קבלה.
4. **security** — בכל קוד שנוגע ב-auth/תשלומים/נתוני משתמש/אדמין/קבצים.
5. **tranzila-payments** — בכל חיבור/שינוי/בדיקה של סליקת אשראי דרך טרנזילה. **קריטיות 10/10** — להפעיל תמיד לצד `security`, לא במקומו.

זרימה תקנית (חובה בסדר הזה): `planning → עדכון ROADMAP → development → security → qa → עדכון ROADMAP (סיום)`.
**תמיד קודם לתכנן, ואז לעדכן את הרודמאפ עם תכנון הפיצ'ר/שלב, ורק אז לבנות.**

## ⚠️ חוק קריטי — עדכון רודמאפ
**הקובץ `ROADMAP.md` הוא מקור האמת להתקדמות.** הוא מתעד מה כבר נעשה ומה נשאר.

לפני סיום כל סשן — **כשהמשתמש נותן הוראה לסיים/לסכם/לעדכן** — חובה:
1. לעדכן ב-`ROADMAP.md` את המשימות שהושלמו (✅), בעבודה (🔄) ושנותרו (⬜).
2. להוסיף רשומה ל**Session Log** בתחתית הרודמאפ (תאריך + מה נעשה + מה הבא).
3. רק אז לבצע commit/push לפי הוראת המשתמש.

אל תעדכן "בוצע" על משימה שלא עברה QA.

## כללי עבודה
- כל טקסט למשתמש בעברית; שמות בקוד באנגלית.
- כסף תמיד ב-integers (אגורות), לעולם לא float.
- אסור לחשוף סודות לקליינט; ENV מאומת ב-`lib/env.ts`.
- בדיקת הרשאות בכל server action; לא לסמוך על ID מהקליינט.
- `npm run lint && npm run typecheck && npm test` ירוקים לפני "סיימתי".
