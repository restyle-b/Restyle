# תכנון: רב-לשוניות (i18n) — עברית / אנגלית / ערבית

## ערך למשתמש
לקוחות שלא קוראים עברית (תיירים, דוברי אנגלית/ערבית) יכולים לגלוש באתר, להבין את
השירותים והאקדמיה, וליצור קשר/לקבוע תור — בלי תלות בעברית.

## החלטות טכניות
- **ספרייה:** `next-intl` — תמיכה רשמית ב-App Router (Next 15), ניתוב לפי `[locale]`,
  `middleware` מובנה, טעינת הודעות לפי locale, `generateMetadata` per-locale.
- **שפות:** `he` (ברירת מחדל), `en`, `ar`.
- **ניתוב:** `localePrefix: "as-needed"` — עברית בלי prefix (`/`, `/about`),
  אנגלית/ערבית עם prefix (`/en`, `/en/about`, `/ar`, `/ar/about`). **אין** auto-redirect
  לפי שפת דפדפן — שורש האתר תמיד עברית כברירת מחדל, לפי דרישת המשתמש.
- **כיווניות:** `<html lang dir>` ב-layout לפי locale — `he`/`ar` → `rtl`, `en` → `ltr`.
  Tailwind כבר RTL-first; אלמנטים עם מיקום מוחלט/קבוע (`right-4`/`left-4`) יעברו ל-
  `inset-inline` (`end-4`/`start-4`) כדי שיתהפכו נכון גם ב-LTR.
- **מבנה תיקיות:** כל ה-routes הציבוריים עוברים מ-`src/app/*` ל-`src/app/[locale]/*`.
  Routes לא-מתורגמים (`sitemap.xml`, `robots.txt`, `icon.svg`) נשארים בשורש.
- **הודעות:** `messages/{he,en,ar}.json` — JSON מקונן לפי עמוד/קומפוננטה. תוכן עברי
  הוא **מקור האמת**; אנגלית/ערבית מתורגמות על ידי Claude בסשן זה — **מומלץ בדיקת
  דובר שפת-אם לפני production**, בייחוד לערבית (ניואנסים תרבותיים/דתיים בענף הספרות).
- **middleware:** שילוב `next-intl` middleware עם ה-middleware הקיים של Supabase
  (session refresh + הגנת `/account`) — next-intl ראשון (קובע locale), אז Supabase.
- **מתג שפה:** קומפוננטת `LocaleSwitcher` ב-header (ובתפריט מובייל) — שומר על העמוד
  הנוכחי, מחליף locale.
- **נתוני תוכן** (`services-data.ts`, `academy-data.ts`, `testimonials-data.ts`):
  מועברים להודעות (`messages/*.json`) כמערכים לפי locale, כדי שיתורגמו בלי לוגיקה כפולה.
- **SEO:** `alternates.languages` (hreflang) ב-metadata כל עמוד; `sitemap.xml` כולל
  גרסאות locale.

## קריטריוני קבלה
- [ ] `/`, `/en`, `/ar` כולם 200, עם `dir`/`lang` נכונים ב-`<html>`.
- [ ] כל טקסט UI (לא תוכן שיווקי שעדיין לא תורגם) מגיע מהודעות locale — אין מחרוזות עבריות קשיחות בקומפוננטות.
- [ ] מתג שפה עובד מכל עמוד, נשאר באותו עמוד.
- [ ] RTL (he/ar) ו-LTR (en) נראים תקין ב-Playwright (header/footer/לחצנים צפים לא הפוכים/חורגים).
- [ ] typecheck/lint/test/build ירוקים.
- [ ] קישורי טלפון/וואטסאפ/וייז ממשיכים לעבוד בכל locale (תוכן ההודעה ב-wa.me מתורגם).

## סדר עבודה
1. התקנת `next-intl`, הקמת `i18n/routing.ts` + `i18n/request.ts` + `i18n/navigation.ts`.
2. שילוב middleware.
3. העברת `src/app/*` ל-`src/app/[locale]/*`.
4. יצירת `messages/he.json` (חילוץ הטקסט הקיים) → תרגום ל-`en.json`/`ar.json`.
5. עדכון כל הקומפוננטות/עמודים לשימוש ב-`useTranslations`/`getTranslations`.
6. `LocaleSwitcher` + עדכון header/footer.
7. תיקון מיקומים מוחלטים ל-logical properties (RTL/LTR).
8. QA: Playwright לשלוש השפות + typecheck/lint/test/build.
